What is DocForge?

DocForge is a web application that solves the problem of missing, incomplete, or inconsistent project documentation. Developers typically skip documentation because it takes too long. DocForge eliminates that friction by offering two paths:

Manual Mode — The user fills in structured forms for each documentation section: Overview, Design System, Frontend, Backend, Database, API Routes, Project Structure, Workflow, Auth, Testing, Deployment, and Extra Features. Every field has helpful labels and hints. Once filled, the AI writes a polished, detailed document for that section.

AI Mode — The user writes one description of their project. The AI generates all sections automatically in one flow — from overview to deployment. The output is streamed live section by section.

The final output is a single self-contained HTML file — a styled, interactive, multi-page documentation website the user can view in any browser, edit directly, add new sections later, and share with teammates.

Core Value Proposition: Zero-friction documentation. Write once, get a professional full-stack project doc in under 60 seconds — complete with DB schema, API routes, design tokens, deployment guide, and more.

Core Problem Solved

Documentation is universally acknowledged as important but universally skipped because:

Takes too long to write No clear structure to follow Gets outdated and ignored Hard to maintain across team Generic templates feel impersonal

DocForge solves all of these by combining structured input (so the user only provides facts, not prose), AI writing (so the language and structure are handled automatically), and editable HTML output (so the doc stays alive and can be updated any time).

Both modes produce the same final output — an editable, downloadable HTML documentation file. The difference is how the content is created: manually with AI-assisted writing per section, or fully automatically from a single description.

Authentication Flow (v2+)

TypeScript — auth middleware
// middleware/auth.ts (Next.js App Router)
import { NextRequest, NextResponse } from 'next/server';
import { verifyJWT } from '@/lib/jwt';

export async function middleware(req: NextRequest) {
  const token = req.cookies.get('auth_token')?.value;

  if (!token) {
    return NextResponse.redirect(new URL('/login', req.url));
  }

  try {
    const payload = await verifyJWT(token);
    const requestHeaders = new Headers(req.headers);
    requestHeaders.set('x-user-id', payload.userId);
    requestHeaders.set('x-user-role', payload.role);

    return NextResponse.next({ request: { headers: requestHeaders } });
  } catch {
    return NextResponse.redirect(new URL('/login', req.url));
  }
}

Gemini Streaming Implementation

TypeScript — streamGemini()
async function streamGemini(
  model: string,
  prompt: string,
  onChunk: (chunk: string, full: string) => void
): Promise<string> {
  const url = `https://generativelanguage.googleapis.com/v1beta/models/${model}` +
              `:streamGenerateContent?key=${getApiKey()}&alt=sse`;

  const response = await fetch(url, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      contents: [{ role: 'user', parts: [{ text: prompt }] }],
      generationConfig: { maxOutputTokens: 8192, temperature: 0.75, topP: 0.95 }
    })
  });

  if (!response.ok) {
    const err = await response.json().catch(() => ({}));
    throw new Error(err?.error?.message || `HTTP ${response.status}`);
  }

  const reader = response.body!.getReader();
  const decoder = new TextDecoder();
  let buffer = '';
  let fullText = '';

  while (true) {
    const { done, value } = await reader.read();
    if (done) break;

    buffer += decoder.decode(value, { stream: true });
    const lines = buffer.split('\n');
    buffer = lines.pop() ?? '';

    for (const line of lines) {
      if (!line.startsWith('data: ')) continue;
      const data = line.slice(6).trim();
      if (!data || data === '[DONE]') continue;
      try {
        const chunk = JSON.parse(data)
          ?.candidates?.[0]?.content?.parts?.[0]?.text ?? '';
        if (chunk) {
          fullText += chunk;
          onChunk(chunk, fullText);
        }
      } catch { /* skip malformed SSE frames */ }
    }
  }
  return fullText;
}

    ${sectionsHTML}
  </div>
  <script>${SIDEBAR_NAVIGATION_JS}</script>
</body>
</html>`;

  const blob = new Blob([html], { type: 'text/html' });
  const url = URL.createObjectURL(blob);
  const a = document.createElement('a');
  a.href = url;
  a.download = `documentation-${Date.now()}.html`;
  a.click();
  URL.revokeObjectURL(url);
}

// middleware/auth.ts
export const tenantGuard = async (req: Request, res: Response, next: NextFunction) => {
  const authHeader = req.headers.authorization;
  if (!authHeader?.startsWith('Bearer ')) {
    return res.status(401).json({ error: 'No token provided' });
  }

  const token = authHeader.slice(7);
  try {
    const payload = await verifyJWT(token);
    req.user = { id: payload.userId, role: payload.role };
    next();
  } catch {
    return res.status(401).json({ error: 'Invalid or expired token' });
  }
};

// Role-based access control
export const requireRole = (roles: string[]) =>
  (req: Request, res: Response, next: NextFunction) => {
    if (!roles.includes(req.user.role)) {
      return res.status(403).json({ error: 'Insufficient permissions' });
    }
    next();
  };

Error Handling

TypeScript — Gemini error types
// Common Gemini API errors and how DocForge handles them:

// 400 Bad Request — malformed prompt or content policy
// → Show user-friendly message, keep form data intact

// 401 Unauthorized — invalid API key
// → Set API key indicator to red, prompt user to check key at aistudio.google.com

// 429 Too Many Requests — rate limit hit
// → Show "Rate limit reached. Wait 60 seconds or switch to Flash Lite model."
//   Display retry countdown timer

// 500 Server Error — Gemini service issue
// → "Google AI is temporarily unavailable. Try again in a moment."

async function handleGeminiError(error: Error, response?: Response): Promise<string> {
  if (response?.status === 401) return 'Invalid API key — check aistudio.google.com';
  if (response?.status === 429) return 'Rate limit hit — wait 60s or switch model';
  if (response?.status === 400) return 'Request blocked — try rephrasing your description';
  return error.message || 'Generation failed — please try again';
}

Export

AI Enhancement

After generating a section, user can click "Regenerate" to get a different AI-written version. Useful when the first output isn't quite right. Each regeneration uses the same form data but a different prompt variation for diversity.
AI Enhancement
AI Edit Mode
Select any text in an editable block and ask the AI to rewrite, expand, shorten, or translate it. An inline prompt bar appears with the selection highlighted. AI rewrites only the selected text in context.
Onboarding

Connect a GitHub repo and DocForge analyzes the codebase (README, package.json, folder structure) to pre-fill documentation forms automatically. AI infers tech stack, features, and architecture from the actual code.
